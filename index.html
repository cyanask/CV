工作方法

01 / 项目问题 业务判断

02 / 证据结构 对象建模

将合同、流水、客户/供应商、收入成本和披露文本映射为证据对象。

03 / 智能体框架 受控路由

按任务路由智能体：检索、比对、起草、复核，并限定事实源和输出边界。

04 / 智能体循环 检查与修订

起草、检查、钩稽、修订，每轮保留差异、依据和人审结论。

05 / 交付物 可审计输出

输出招股书、问询回复支持、底稿、研究材料和验证报告。

问题 → 证据对象 → 受控智能体 → 一致性检查 → 人审 → 交付物
